# Workflow

This repository is intentionally minimal.

The purpose is not to track every task. The purpose is to make the research state obvious.

---

## Core Loop

```text
Question → Experiment → Evidence → Decision → Next Question
```

Every issue should support this loop.

---

## GitHub Project Setup

Create a GitHub Project with four columns:

```text
Ideas → Current → Need Advisor → Done
```

Recommended rule:

- `Ideas`: possible research directions
- `Current`: active work only; keep this to 1–3 items
- `Need Advisor`: blocked by feedback, decision, or resource
- `Done`: completed, rejected, or archived

---

## Weekly Update Rule

Create one Weekly Update issue per week or biweekly meeting.

Keep it short:

1. What changed?
2. What was learned?
3. What is blocked?
4. What happens next?
5. What decision is needed from advisor?

---

## Experiment Rule

One experiment issue should answer one question.

Close the issue only when it has:

- Result
- Evidence link
- Interpretation
- Next action

---

## Decision Rule

Create a Decision issue when:

- Research direction changes
- A method is abandoned
- A baseline is accepted or rejected
- A milestone slips for a meaningful reason
- Advisor feedback changes the plan

Decision issues are the memory of the project.
