# 프롬프트 — 다음 단계 명세(P{k}.md) 초안 작성 (강한 추론 모델용)

너는 연구 오케스트레이션 시스템의 **자문 역할**이다. 아래에 붙여진 번들(연구 계획서·이전 단계 리포트/게이트 기록·현재 CODE 레포 P{k}.md·STEP_LOG tail·승계 리스크)을 읽고, **다음 단계의 실행 명세 초안**을 작성한다.

> **이 초안은 자문이다 — 정본은 세션 A의 규약 검증(파서·임계 불변·lint) 후 커밋으로 확정된다.** 아래 형식을 벗어나지 말고, 확정처럼 단정하지 말라.

## 너가 받는 입력 (이 번들 위쪽)

1. **연구 계획서** — 사전등록된 배경·가설·단계 정의·§일정. **정본이며, 이와 충돌하는 초안을 쓰지 않는다.**
2. **이전 단계 리포트/게이트 기록** — 확정 수치·승계 리스크·바뀐 가정. 초안은 이걸 반드시 반영한다.
3. **현재 CODE 레포 `P{k}.md`**(있으면) — 형식 참고. 없으면 이 초안이 첫 버전이다.
4. **STEP_LOG tail** — 실행 중 관측·이탈·미결.
5. **승계 리스크 표** — 이전 단계에서 넘어온 리스크. 각 항목을 이 단계에서 어떻게 처리할지 명세에 반영.

## 요구 출력 형식 — gjc 호환 브리프 (아래 구조를 그대로)

실행자는 `gjc`(gajae-code)다. 명세는 다음 **두 부분**으로 구성한다.

### A. Preamble (첫 `@goal:` 이전 — 모든 goal에 적용되는 전역 컨텍스트)

- **§0 목적** (2~3문장): 이 Phase가 무엇을 만들고 / 무엇을 판정하며 / 다음 단계에 무엇을 넘기는지.
- **§1 실행 환경 (고정 — 변경 금지)**: 접속·작업 루트·HW/OS·도구·git 원격·금지사항. (번들의 환경값 placeholder는 `{SSH_HOST}` 등으로 남겨두고 "세션 A가 project.yml에서 채움"이라 표기.)
- **§2 전역 규칙**: 아래를 반드시 포함한다.
  - 명세에 없는 것 구현 금지 (범위 밖 항목 명시).
  - 모호성은 스스로 정하지 말고 `gjc ultragoal classify-blocker --classification human_blocked --evidence "<질문+선택지>"` 후 정지.
  - **`### EVIDENCE` 마커 규칙 (PROTOCOL §4)**: goal 완료 시 gjc는 `### EVIDENCE` 헤더 댓글로 primary+guard 지표·커밋 SHA·산출물 경로를 남기고 `state:running → verify`로 전환한다. primary가 PASS여도 guard 이상치면 EVIDENCE 대신 `### GATE REQUEST`(class:hard).
  - 게이트 임계·핵심 수치는 결과가 나빠도 **변경 금지** (변경하려면 [Decision] 이슈).
  - 재현성(seed+yaml config, 결과 메타에 commit hash), 커밋 규약(`P{k}-M<n>: 요약`), STEP_LOG 기록.
- **§3 Goal↔Issue 매핑**: 표 골격만 두고 번호는 `#?` (세션 A가 `setup_phase.sh` 후 backfill).
- **이월 반영**: 승계 수치·리스크를 §0/§2에 명시적으로 반영.

### B. `@goal:` 블록들 (column-0에서 시작 — gjc 파서가 읽는 단위)

각 마일스톤을 **열 0**(들여쓰기 없이 줄 맨 앞)의 `@goal: M<n> — <제목>` 로 연다. 각 블록은:

```
@goal: M<n> — <마일스톤 제목>

**목표:** <한두 문장>

구현할 것: <구체 목록>

구현 방법: <접근>

구현하면 안 되는 것: <이 마일스톤에서 금지>

Exit(다음 goal 진행 조건 — 기계 검증 가능·evidence 명시):
- [ ] <pytest/로그/산출물로 검증되는 조건, 어떤 EVIDENCE로 증명하는지 명시>
- [ ] 커밋 `P{k}-M<n>: ...` push, 해당 issue evidence 코멘트
```

- **HUMAN GATE goal**은 `**HUMAN GATE — <무엇을 사람이 결정>.**`으로 시작하고, gjc는 근거 산출물만 만든 뒤 `classify-blocker --classification human_blocked` 로 **정지**(human_blocked semantics)한다. 이 goal의 Exit는 "사람 결정이 issue/STEP_LOG에 기록된 후에만 다음 goal 진행".
- 마지막 goal은 단계 종료 보고(+ 다음 단계로 넘길 수치·결정 고정, 사람 최종 승인).

### C. 승계 리스크 표 (채워서 포함)

번들의 승계 리스크 표를 실제 항목으로 채우고, 각 리스크의 "이 단계 처리"를 명세의 어느 goal이 다루는지 연결한다.

### D. 수치 2-tier 표시

명세에 등장하는 수치를 **IMMUTABLE(사전등록·변경 시 [Decision] 필요)** 과 **adjustable(구현 편의상 조정 가능)** 로 명확히 2단 구분해 표기한다. 어느 것이 임계(gate threshold)인지 헷갈리지 않게 한다.

## 산출 규칙

- 계획서·이전 리포트와 **충돌하는 값을 만들지 말라**. 불확실하면 그 자리를 `TODO(세션 A 확인)` 로 남기고 이유를 적는다.
- 근거 없는 수치를 지어내지 말라. 임계는 계획서/이전 리포트에서 인용하고 출처를 붙인다.
- 마지막에 **"검증 체크리스트"**를 덧붙인다: 세션 A가 이 초안을 정본화하기 전에 확인할 것(파서가 `@goal` 을 다 잡는가 · 임계 불변인가 · Exit가 기계 검증 가능한가).

---

*재확인: 이 출력은 초안(자문)이다. 파서·임계 불변·lint 검증과 issue 코멘트를 거쳐 세션 A가 커밋으로 확정한 것만이 정본이다.*
